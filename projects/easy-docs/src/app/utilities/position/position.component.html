Position

Postition classes are used to determine where an element should be placed in the viewport or another element.

These classes allow for the use of bottom, left, right, and top properties in CSS.

Note: Caution should be used when deciding to use position as the results can be unpredicatable. Flexbox or grid can be used instead yielding more predicatable results.

Asolute

Use a .pos-a class to fix an element absolutely to a relatively positioned containing element.

Note: Without a relatively positioned containing element, the position absolute element will be fixed absolutely to its closest ancestor (the body element).

absolute with left: 15rem (240px) from the body

Fixed

Use a .pos-f-[b || l || r || t] class to fix an element to the bottom, left, right, or top of the viewport, respectively.

Note: Padding is needed on other elements to prevent them from overlapping the fixed one.

Relative

Use a .pos-r class to create a relative element container for a child position absolute element.

Sticky

Use a .pos-s class to make an element relatively positioned until a top value is reached on the body.

After the top value is reached on the body, the element becomes positioned fixed.

Note: Position sticky only works on "evergreen" browsers (Chrome, Edge > 15, Firefox, and Safari).
